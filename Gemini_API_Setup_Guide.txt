STEP 1 — Open Google AI Studio
───────────────────────────────
  1. Open your browser.
  2. Go to this exact URL:

        https://aistudio.google.com

  3. You will see the Google AI Studio homepage.
     NOTE: If you are in India, this works fine. Gemini API is available
     in India on the free tier.

──────────────────────────────────────────────────────────────────────────────

STEP 2 — Sign In With Your Google Account
───────────────────────────────────────────
  1. Click "Sign In" (top right corner).
  2. Use any Gmail account — your personal Gmail is fine.
  3. First-time users: you will be shown Google's AI Studio
     Terms of Service.
     → Read them, then click "I Agree" / "Accept" to continue.

  ⚠️  IMPORTANT: Do NOT use your college or institution Google account
      if it has restrictions. Use a personal Gmail account.

──────────────────────────────────────────────────────────────────────────────

STEP 3 — Go to the API Keys Page
──────────────────────────────────
  Option A (Quickest):
    → Direct link: https://aistudio.google.com/app/apikey

  Option B (Manual):
    1. Look at the LEFT SIDEBAR of Google AI Studio.
    2. Click "Get API key" (it has a key icon 🔑 next to it).
    3. This opens the API Keys management page.

──────────────────────────────────────────────────────────────────────────────

STEP 4 — Create a New API Key
───────────────────────────────
  1. On the API Keys page, click the blue button:
        [ + Create API key ]

  2. A dialog box appears with two options:
        Option A → "Create API key in new project"  ← CHOOSE THIS
        Option B → "Create API key in existing project"

  3. Click "Create API key in new project".

  4. Google automatically:
        - Creates a new Google Cloud project for you
        - Generates an API key linked to that project

  5. A box appears showing your key. It will look like this:

        AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

     (Always starts with "AIza" followed by random characters)

──────────────────────────────────────────────────────────────────────────────

STEP 5 — Copy and Save Your Key
─────────────────────────────────
  1. Click the COPY button (📋 icon) next to your key.

  2. Open a temporary plain text file on your computer (Notepad,
     VS Code, or any editor) and paste the key there temporarily.
     Example:
        My Gemini Key: AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

  3. DO NOT close the AI Studio tab yet — keep it open.

  ⚠️  SECURITY RULES — READ THESE:
      ❌ Never paste your key in any chat (WhatsApp, Telegram, etc.)
      ❌ Never commit it to GitHub / push it in any file
      ❌ Never share it with anyone outside your team
      ✅ Only store it in .env files (explained in Part 2 below)

STEP 7 — Create the Backend .env File
───────────────────────────────────────
  1. Open your code editor (VS Code recommended).
  2. Navigate to your backend folder:
        ez-dnd/backend/

  3. Check if a file named ".env" already exists.
     - If YES → open it and add the lines below.
     - If NO  → create a new file named exactly:  .env
               (just a dot followed by "env", no other extension)

  4. Add the following lines to your backend/.env file:

  ┌─────────────────────────────────────────────────────────────────┐
  │  # ── AI Code Generation ───────────────────────────────────── │
  │  GEMINI_API_KEY=AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX        │
  │                                                                 │
  │  # ── Deployment ────────────────────────────────────────────  │
  │  NETLIFY_TOKEN=your_netlify_token_here                          │
  │                                                                 │
  │  # ── Database ──────────────────────────────────────────────  │
  │  MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net│
  │                                                                 │
  │  # ── Authentication ────────────────────────────────────────  │
  │  FIREBASE_PROJECT_ID=your_firebase_project_id                   │
  │  FIREBASE_PRIVATE_KEY=your_firebase_private_key                 │
  │  FIREBASE_CLIENT_EMAIL=your_firebase_client_email               │
  │                                                                 │
  │  # ── Server ────────────────────────────────────────────────  │
  │  PORT=5000                                                      │
  │  NODE_ENV=development                                           │
  │                                                                 │
  │  # ── Fallback AI (Groq) ────────────────────────────────────  │
  │  GROQ_API_KEY=your_groq_key_here                                │
  └─────────────────────────────────────────────────────────────────┘

  Replace AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
  with your actual key copied in Step 5.

  ⚠️  FORMATTING RULES FOR .env FILES:
      ✅ No spaces around the = sign
      ✅ No quotes around the value (unless the value has spaces)
      ✅ No trailing spaces after the value
      ✅ Each variable on its own line
      ❌ GEMINI_API_KEY = AIzaSy...   (WRONG — spaces around =)
      ❌ GEMINI_API_KEY="AIzaSy..."   (WRONG — quotes not needed)
      ✅ GEMINI_API_KEY=AIzaSy...     (CORRECT)

STEP 9 — Install dotenv in Your Backend
─────────────────────────────────────────
  The dotenv package reads your .env file and loads the variables
  into your Node.js app automatically.

  In your terminal, navigate to the backend folder and run:

        cd ez-dnd/backend
        npm install dotenv

  Then, at the VERY TOP of your backend's main file (server.js
  or index.js), add this as the FIRST LINE:

        require('dotenv').config();

  Example server.js structure:
  ─────────────────────────────
        require('dotenv').config();   // ← Must be first line

        const express = require('express');
        const app = express();

        // Now you can access your key anywhere:
        const geminiKey = process.env.GEMINI_API_KEY;
        const netlifyToken = process.env.NETLIFY_TOKEN;

        app.listen(process.env.PORT || 5000);

STEP 10 — Protect Your Key With .gitignore
────────────────────────────────────────────
  This is CRITICAL. Without this step, your key could get
  accidentally pushed to GitHub and stolen within minutes by bots.

  1. In your project ROOT folder (ez-dnd/), open or create
     a file named exactly:  .gitignore

  2. Add these lines to it:

  ┌─────────────────────────────────────────────────────────────────┐
  │  # Environment variable files — NEVER commit these             │
  │  .env                                                           │
  │  .env.local                                                     │
  │  .env.development                                               │
  │  .env.production                                                │
  │  backend/.env                                                   │
  │  frontend/.env                                                  │
  │                                                                 │
  │  # Node modules                                                 │
  │  node_modules/                                                  │
  └─────────────────────────────────────────────────────────────────┘

  3. Save the file.

  4. If you already have a Git repo, run this command to make sure
     Git stops tracking any .env files that were already added:

        git rm --cached backend/.env
        git rm --cached frontend/.env

  ✅ From now on, even if you run "git push", your keys stay safe.

  PART 3 — USING THE KEY IN YOUR BACKEND CODE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Create this file in your backend:
  ez-dnd/backend/services/aiService.js
  ─────────────────────────────────────

        require('dotenv').config();

        // ── Primary: Gemini 2.5 Flash-Lite ─────────────────────────────
        async function generateWithGemini(prompt) {
          const url = `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash-lite:generateContent?key=${process.env.GEMINI_API_KEY}`;

          const response = await fetch(url, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
              contents: [{
                parts: [{ text: prompt }]
              }],
              generationConfig: {
                temperature: 0.7,
                maxOutputTokens: 2048,
              }
            })
          });

          if (!response.ok) {
            const error = await response.json();
            throw { status: response.status, error };
          }

          const data = await response.json();
          return data.candidates[0].content.parts[0].text;
        }

        // ── Fallback: Groq (Llama 3) ─────────────────────────────────
        async function generateWithGroq(prompt) {
          const response = await fetch('https://api.groq.com/openai/v1/chat/completions', {
            method: 'POST',
            headers: {
              'Authorization': `Bearer ${process.env.GROQ_API_KEY}`,
              'Content-Type': 'application/json'
            },
            body: JSON.stringify({
              model: 'llama-3.3-70b-versatile',
              messages: [
                {
                  role: 'system',
                  content: 'You are a frontend code generator. Output ONLY raw HTML+CSS+JS. No markdown. No explanation. No code fences.'
                },
                { role: 'user', content: prompt }
              ],
              max_tokens: 2048,
              temperature: 0.7,
            })
          });

          const data = await response.json();
          return data.choices[0].message.content;
        }

        // ── Main function with automatic fallback ───────────────────
        async function generateComponent(userPrompt) {
          const systemPrompt = `
            You are an expert frontend developer.
            Generate a responsive HTML component based on the user's description.
            Rules:
            - Output ONLY the raw HTML block (no <html>, <head>, or <body> tags)
            - Include all CSS inside a <style> tag within the component
            - Use modern, clean design with subtle shadows and rounded corners
            - Make it mobile-responsive
            - Use the color scheme: teal (#0D9488) and blue (#2563EB) as accents
            - No JavaScript unless the user specifically asks for it
            - No external library imports (no Bootstrap, no Tailwind CDN)

            User request: ${userPrompt}
          `;

          try {
            console.log('Trying Gemini 2.5 Flash-Lite...');
            const result = await generateWithGemini(systemPrompt);
            console.log('Gemini succeeded ✅');
            return { html: result, source: 'gemini' };
          } catch (err) {
            if (err.status === 429) {
              console.log('Gemini rate limited. Switching to Groq...');
              try {
                const result = await generateWithGroq(systemPrompt);
                console.log('Groq succeeded ✅');
                return { html: result, source: 'groq' };
              } catch (groqErr) {
                throw new Error('Both AI providers failed. Try again in a moment.');
              }
            }
            throw err;
          }
        }

        module.exports = { generateComponent };


  Then in your Express router (routes/ai.js):
  ─────────────────────────────────────────────

        const express = require('express');
        const router = express.Router();
        const { generateComponent } = require('../services/aiService');

        router.post('/generate-component', async (req, res) => {
          try {
            const { prompt } = req.body;

            if (!prompt || prompt.trim().length === 0) {
              return res.status(400).json({ error: 'Prompt is required' });
            }

            const result = await generateComponent(prompt);
            res.json({ success: true, html: result.html, source: result.source });

          } catch (err) {
            console.error('AI generation error:', err);
            res.status(500).json({ error: err.message || 'Generation failed' });
          }
        });

        module.exports = router;

  PART 4 — DEPLOYING TO RENDER / RAILWAY (PRODUCTION .ENV)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  When you deploy your backend to Render or Railway, you do NOT
  upload the .env file. Instead, you enter the variables in
  the platform's dashboard. Here's how:

  ── For Render (render.com) ─────────────────────────────────────

    1. Go to render.com and open your backend service.
    2. Click "Environment" in the left sidebar.
    3. Click "Add Environment Variable".
    4. Add each variable one by one:

         Key: GEMINI_API_KEY
         Value: AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

         Key: NETLIFY_TOKEN
         Value: your_netlify_token

         Key: MONGODB_URI
         Value: mongodb+srv://...

         Key: GROQ_API_KEY
         Value: gsk_XXXXXXXXXXXXXXXXXX

    5. Click "Save Changes".
    6. Render will automatically restart your service with the new keys.

  ── For Railway (railway.app) ───────────────────────────────────

    1. Go to railway.app and open your project.
    2. Click on your backend service.
    3. Go to the "Variables" tab.
    4. Click "+ New Variable" and add each key-value pair.
    5. Railway restarts automatically.

  ── For Vercel (frontend only) ──────────────────────────────────

    1. Go to vercel.com and open your frontend project.
    2. Go to Settings → Environment Variables.
    3. Add only VITE_ prefixed variables:

         VITE_API_URL = https://your-backend.onrender.com
         VITE_FIREBASE_API_KEY = your_firebase_web_api_key
         VITE_FIREBASE_AUTH_DOMAIN = your_project.firebaseapp.com
         VITE_FIREBASE_PROJECT_ID = your_project_id

    4. Click Save. Redeploy for changes to take effect.

  TROUBLESHOOTING — COMMON ERRORS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ERROR: "API_KEY_INVALID"
  ─────────────────────────
  Cause:  Key was copied incorrectly (extra space or missing chars).
  Fix:    Go back to aistudio.google.com/app/apikey and recopy it.
          Paste into a plain text editor first to check for spaces.

  ERROR: "PERMISSION_DENIED"
  ───────────────────────────
  Cause:  The Generative Language API is not enabled on your project.
  Fix:    Go to: https://console.cloud.google.com/apis/library
          Search for "Generative Language API" and click Enable.

  ERROR: 429 "RESOURCE_EXHAUSTED"
  ─────────────────────────────────
  Cause:  You hit the free tier rate limit (10-15 requests/minute).
  Fix:    Your fallback to Groq should handle this automatically.
          If testing manually, wait 60 seconds and try again.

  ERROR: process.env.GEMINI_API_KEY is undefined
  ───────────────────────────────────────────────
  Cause:  dotenv not loaded, or .env file in wrong location.
  Fix 1:  Make sure require('dotenv').config() is the FIRST line
          in your server.js / index.js.
  Fix 2:  Make sure your .env file is in the same folder as
          your package.json (i.e., ez-dnd/backend/.env).
  Fix 3:  Restart your Node.js server after editing .env.

  ERROR: Key works in terminal but not in deployed app
  ──────────────────────────────────────────────────────
  Cause:  .env file is not uploaded to the server (correct!).
          But you also haven't added the key in Render/Railway.
  Fix:    Go to your Render/Railway dashboard and add the
          environment variable there (see Part 4 above).
